AJAX、跨域

AJAX概念：

Asynchronous JavaScript and XML（异步JavaScript和XML)

作用：

不重新加载整个页面的情况下，可以与服务器交换数据并更新部分网页内容。 

节省用户操作，时间，提高用户体验，减少数据请求传输获取数据

过程：

初始化 – >发送请求 –> 服务器收到请求 –> 服务器处理 –> 返回结果

表单

  • 表单form

    • action 提交到哪里

    • method 提交方式

    • 第一个： 发送方式 // GET POST

    • 第二个： 发送地址 // 发送到的地址

    • 第三个： 发送内容

  • Get方式

    • Get通过url地址传输

    • GET请求 发送数据大小有限制；

    • 提交内容是明文的，不安全

    • GET 请求传输速度快；

  • Post方式

    • Post通过浏览器内部传输

    • 数据量理论上无限

    • 相对比较安全

AJAX发送方式：

GET请求：

var  xhr=new XMLHttpRequest();  //初始化一个 xmlhttpqequest对象

xhr.open("get","/logo?username="+user_val+"&psd="+psd_val); 

注： 有三个参数：
                  第一个： 请求方式

                  第二个： 请求地址

                  第三个： 同步还是异步

            （布尔值，true：异步；false：同步；默认异步）

xhr.send(); 

xhr.onreadystatechange = function () {}

        注： 当我们的ajax 对象状态发生改变的时候，请求状态发生改变

var data = xhr.responseText;

readyState属性：请求状态

data = 0 （初始化）还没有调用open()方法

data = 1（载入）已调用send()方法，正在发送请求

data = 2（载入完成）send()方法完成，已收到全部响应内容

data = 3（解析）正在解析响应内容

data = 4（完成）响应内容解析完成，可以在客户端调用了

    注：如果想要操作后台数据的话，必须是在data = 4时才可以

作用：

    xhr.responseXML：获取后台发来的XML 格式的数据；

跨域：

同源策略：就是只允许访问当前网站的资源，不允许访问其他网站的资源，这样就更安全

何为同源？

协议   域名   端口 一致就是同源，有一项不同就是跨域

跨域请求：

浏览器虽然不允许 对外部的网站发送请求，但是允许我们 请求图片。js，等资源

所以利用这一特性，我们可以进行跨域发送数据，获取数据

操作：

1. 动态创建一个带有src 属性的标签。 （一般正常情况下使用script）；

2. 让 这个标签的src 属性等于我们要请求的地址；

3. 把这个标签放到页面里面；

4. 获取数据操作数据；
